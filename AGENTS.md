# Правила работы с Bogobot Static

## Источник проекта

- Редактируемый источник сайта: `work/bogobot-static`.
- Публикуемая статическая версия: `outputs/bogobot-prototype`.
- Obsidian-архив `05_Богобот` является каноническим источником текстов, связей, метаданных и визуальных материалов.
- Quartz-проект `work/bogobot-prototype` — отдельный устаревший прототип. Не изменять его в рамках работы над этим сайтом.
- Не перечитывать весь Obsidian-архив при каждой задаче. Сначала использовать документы в `docs/`, затем открывать только нужные исходные файлы.

## Канон и граф

- Одна сущность мира соответствует одному основному узлу графа.
- Синтетические и редакционно собранные тексты не маркируются как исходный канон.
- Четвёртый уровень `archive` пока хранится только в данных и не выводится визуально.

## Интерфейс

- Не создавать локальные CSS-костыли, если решение относится к общему компоненту или общей переменной.
- Синий используется только для `active`, `selected`, `hover`, `focus`, активных формул и связей.
- Изображения и реликвии выравниваются по общей левой оси.
- Архивные примечания используют один общий компонент.

## Публикация

- Не редактировать `outputs/bogobot-prototype` вручную.
- Все изменения вносить только в `work/bogobot-static`.
- После изменений запускать `npm run publish`.
- Скрипт публикации не должен затрагивать резервные копии, Quartz или Obsidian-архив.

## Порядок чтения контекста

1. `docs/STATE.md`
2. `docs/DECISIONS.md`
3. `docs/SITE_PLAN.md`
4. Нужная строка в `docs/CANON_MAP.md` или `docs/IMAGE_MAP.md`
5. Только необходимые файлы сайта и Obsidian
