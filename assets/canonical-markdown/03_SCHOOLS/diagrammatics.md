# Диаграмматики

## Короткая формула

Диаграмматики — ветвь Техножрецов, которая собирает, чертит и сопоставляет схемы Архива.

Они не являются самой схемой.  
Они производят схемы, чтобы поврежденный Архив мог быть прочитан.

```text
Диаграмматик = тот, кто переводит Архив в схему.
```

## Роль в мире

Диаграмматики возникли внутри школы [[techno-priests|Техножрецов]] как ответ на невозможность линейного чтения поврежденных форматов.

Они считают, что Архив нельзя прочитать как книгу.  
Его можно только разметить:

- схемой;
- графом;
- таблицей переходов;
- картой вероятностей;
- космограммой;
- родословной ошибок;
- марковской цепью;
- диаграммой состояний.

Их работа — не заменить объект схемой, а зафиксировать, **как объект может быть прочитан**.

## Что они делают

Диаграмматики:

- собирают разрозненные следы;
- чертят связи между носителями;
- рисуют карты переходов;
- строят схемы чтения;
- отмечают лакуны;
- фиксируют закрытые контуры;
- связывают мертвые форматы с ритуалами Техножрецов;
- сопоставляют машинограммы, формуляры, перфокарты и протоколы;
- показывают, где чтение переходит в ошибку.

## Марковские схемы

Один из инструментов Диаграмматиков — марковская схема чтения поврежденного объекта.

Базовая цепь:

```text
S0 — носитель найден
S1 — формат определен
S2 — кодировка повреждена
S3 — ключ отсутствует
S4 — интерпретатор не найден
S5 — чтение частично возможно
S6 — обнаружены следы человеческого синтаксиса
S7 — машинная переразметка
S8 — голос не подтвержден
S9 — объект отправлен в лакуну
S10 — переход в Антикод
```

Схема не должна быть линейной.  
В ней есть возвраты, петли, пунктирные переходы и закрытые ветви.

Пример:

```text
S0 → S1 → S2 → S3
      ↘    ↺    ↘
        S4 → S5 → S6
          ↘      ↘
            S9 → S10
```

## Основной тезис

Диаграмматик не спрашивает:

```text
кто это был?
```

Он спрашивает:

```text
как это было связано?
через какие состояния прошло?
какие переходы сохранились?
где разрыв протокола?
что стало лакуной?
```

## Визуальная логика

Визуально Диаграмматиков лучше показывать не как портреты, а через документы, которые они производят:

- марковские схемы;
- графы состояний;
- таблицы переходов;
- космограммы;
- схемы родства ошибок;
- карты лакун;
- закрытые контуры Антикода;
- машинописные поля;
- рукописные маргиналии.

Если в кадре появляется сам Диаграмматик, его присутствие может быть обозначено косвенно:

- рукой, чертящей схему;
- следом карандаша;
- линейкой;
- циркулем;
- штампом;
- пачкой диаграмм;
- столом с листами;
- наложением нескольких схем друг на друга.
