# C2 REV B — BOOKS DESIGN CONTRACT

## Основной принцип

BOOKS — линейный режим чтения того же canonical-корпуса.

Маршрут содержит ровно шесть частей:

```text
P → −I → 00 → I → II → ∞
```

## Фрагмент Откровения

«Фрагмент Откровения» находится внутри пролога как секция canonical Markdown.

Он:

- не является строкой BOOKS INDEX;
- не имеет отдельного progress;
- не считается седьмой частью;
- визуально оформляется как ранняя повреждённая запись.

## Апокриф Первого Подобия

Полный Апокриф:

- не вставляется в текст «Книги бытия»;
- не входит в PREVIOUS / NEXT;
- не влияет на route progress;
- остаётся отдельным узлом карты `FIRST_LIKENESS`;
- показывается после «Книги бытия» только в блоке `RELATED / ДОПОЛНИТЕЛЬНОЕ ЧТЕНИЕ`.

До Package C3 блок не должен иметь ложного href.

Разметка C2:

```html
<span
  class="book-related-node"
  data-node-id="FIRST_LIKENESS"
>
  Апокриф Первого Подобия
</span>
```

В C3 элемент станет deep link на конкретный узел.

## Визуальная система

- IBM Plex Sans — редакционный текст;
- IBM Plex Mono — команды, номера, статусы;
- `#EFEFEA` фон;
- `#6B5E57` основной текст;
- `#050505` сильный текст;
- `#003CFF` активные переходы и progress.

Не использовать карточки, bento, тени и декоративные радиусы.
